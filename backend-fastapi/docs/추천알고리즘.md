# 식단 추천 알고리즘 설계

## 개요

사용자의 신체 정보, 건강 목표, 질환, 식습관 데이터를 기반으로 영양소 갭을 분석하고 최적의 음식을 스코어링하여 추천하는 알고리즘.

---

## 1단계: TDEE & 목표 영양소 계산

### TDEE 계산 (Mifflin-St Jeor)

검증된 공식, 실측치 대비 ±10% 이내 정확도

```
남성 BMR = (10 × 체중kg) + (6.25 × 신장cm) - (5 × 나이) + 5
여성 BMR = (10 × 체중kg) + (6.25 × 신장cm) - (5 × 나이) - 161

TDEE = BMR × 활동계수
```

| activityLevel      | 계수  | 설명                     |
| ------------------ | ----- | ------------------------ |
| SEDENTARY          | 1.2   | 좌식 생활                |
| LIGHTLY_ACTIVE     | 1.375 | 주 1-3회 가벼운 운동     |
| MODERATELY_ACTIVE  | 1.55  | 주 3-5회 중등도 운동     |
| VERY_ACTIVE        | 1.725 | 주 6-7회 고강도 운동     |

> 출처: Mifflin MD et al. "A new predictive equation for resting energy expenditure in healthy individuals." Am J Clin Nutr. 1990

### 건강목표별 칼로리 조정

| healthGoal    | 조정              | 설명                                  |
| ------------- | ----------------- | ------------------------------------- |
| DIET          | TDEE × 0.80       | 20% 적자 (주당 ~0.5kg 감량 페이스)    |
| LEAN_MASS_UP  | TDEE × 1.10       | 10% 잉여 (체지방 최소화하며 근성장)   |
| BULK_UP       | TDEE × 1.20       | 20% 잉여                              |

### 건강목표별 영양소 비율

| healthGoal    | 단백질 (g/kg/day) | 탄 : 단 : 지 비율 |
| ------------- | ----------------- | ------------------ |
| DIET          | 2.0               | 40 : 30 : 30      |
| LEAN_MASS_UP  | 1.8               | 45 : 30 : 25      |
| BULK_UP       | 1.6               | 50 : 25 : 25      |

단백질은 체중 기반으로 먼저 계산한 뒤 나머지 칼로리를 탄수화물과 지방 비율로 분배한다.

> 출처: ISSN Position Stand — 운동하는 성인의 최적 단백질 섭취량 1.4-2.0 g/kg/day (Jager et al., 2017). 다이어트 시 근손실 방지를 위해 상한인 2.0 g/kg 권장 (Helms et al., 2014).

## 2단계: 끼니별 목표 배분

사용자의 식사 패턴에 따라 일일 목표를 끼니별로 분배

### mealPattern별 칼로리 분배율

| mealPattern              | 아침 | 점심 | 저녁 | 간식 |
| ------------------------ | ---- | ---- | ---- | ---- |
| THREE_MEALS              | 25%  | 35%  | 30%  | 10%  |
| TWO_MEALS                | -    | 50%  | 40%  | 10%  |
| INTERMITTENT_FASTING     | -    | 45%  | 45%  | 10%  |
| MULTIPLE_SMALL_MEALS     | 20%  | 25%  | 25%  | 30%  |

요청된 mealType에 해당하는 비율로 해당 끼니의 목표 영양소를 계산

```
meal_target_calories = daily_target_calories × meal_ratio[mealType]
meal_target_protein  = daily_target_protein  × meal_ratio[mealType]
meal_target_carbs    = daily_target_carbs    × meal_ratio[mealType]
meal_target_fat      = daily_target_fat      × meal_ratio[mealType]
```

> 출처: Young et al. "Distribution of energy intake across the day and weight loss: A systematic review and meta-analysis." Obesity Reviews, 2023. 일찍 먹는 패턴이 같은 칼로리에서 2.5배 체중 감량 효과.

## 3단계: 질환별 필터 & 제약조건

### 하드 필터 (후보 제외)

| disease  | 제외 대상                          | 출처                          |
| -------- | ---------------------------------- | ----------------------------- |
| ALLERGY  | 사용자 지정 알레르기 식재료 포함 음식 | -                             |
| GOUT     | 고퓨린 식품 (내장류, 특정 해산물)  | Mayo Clinic Gout Diet Guide   |

### 소프트 제약 (스코어링에서 감점)

| disease         | 제약 내용                           | 임계값 기준                    | 출처                                  |
| --------------- | ----------------------------------- | ------------------------------ | ------------------------------------- |
| DIABETES        | 고당류 식품 감점, 식이섬유 식품 가점 | 당류 비율 > 30% 시 감점        | ADA Dietary Guidelines, 2023          |
| HYPERTENSION    | 고나트륨 식품 감점                  | 나트륨 > 600mg/1회 제공량      | DASH Diet (NKF)                       |
| HYPERLIPIDEMIA  | 고포화지방/고콜레스테롤 감점        | 포화지방 > 5g/1회 제공량       | AHA Guidelines                        |
| KIDNEY_DISEASE  | 고단백/고칼륨/고인 식품 감점        | 단백질 목표를 0.6-0.8g/kg 하향 | KDOQI 2020                            |
| LIVER_DISEASE   | 고나트륨 감점                       | 나트륨 제한                    | -                                     |
| THYROID_DISEASE | 요오드/셀레늄 함유 식품 가점        | 요오드 150mcg/일, 셀레늄 55mcg/일 | PMC Nutrition & Thyroid Function, 2024 |

신장질환자 한정: 신장질환 사용자는 단백질 목표 자체를 0.8g/kg로 오버라이드한다.

> 출처:
> - KDOQI Clinical Practice Guideline for Nutrition in CKD: 2020 Update (Am J Kidney Dis)
> - Diabetes: 탄수화물 50-60%, 식이섬유 30-50g/day 권장 (PMC, 2023)
> - 고혈압: 나트륨 < 2300mg/day, 이상적으로 < 1500mg/day (NKF DASH)
> - 고지혈증: 포화지방 < 7% of calories, 콜레스테롤 < 200mg/day (AHA TLC)
> - 통풍: 엄격한 퓨린 제한 시 요산 15-20% 감소 (PMC, 2022)

## 4단계: 스코어링 (0-100)

### 점수 구성

```
total_score = gap_score(50) + goal_score(15) + disease_score(15) + preference_score(10) + feedback_score(10)
```

| 항목              | 배점 | 역할                        |
| ----------------- | ---- | --------------------------- |
| Gap Match Score   | 50   | 부족한 영양소를 잘 채우는가 |
| Goal Score        | 15   | 건강목표에 부합하는가       |
| Disease Score     | 15   | 질환 제약에 적합한가        |
| Preference Score  | 10   | 사용자 선호에 맞는가        |
| Feedback Score    | 10   | 과거 피드백 반영            |

---

### 3-1. Gap Match Score (최대 50점)

메인 컨셉: 각 영양소(칼로리, 단백질, 탄수화물, 지방)에 대해 음식이 부족분을 얼마나 채워주는지 계산하고, 가장 부족한 영양소에 더 높은 가중치를 부여한다.

각 영양소별로:
- 부족 상태: 부족분 대비 음식의 기여율 (fill_rate)을 0~1로 계산. 초과 시 1로 클리핑.
- 충분 상태: 이미 충족된 영양소를 많이 가진 음식은 감점.

가중치는 부족 비율 기반 softmax로 정규화한다. 예를 들어 단백질이 65% 부족이고 탄수화물이 50% 부족이면, 단백질을 잘 채워주는 음식에 더 높은 점수가 부여된다.

---

### 3-2. Goal Alignment Score (최대 15점)

칼로리 밀도 (kcal/g)와 단백질 비율을 기준으로 건강목표 정합성을 평가한다.

| healthGoal    | 가점 조건                               | 감점 조건               |
| ------------- | --------------------------------------- | ----------------------- |
| DIET          | 칼로리 밀도 < 1.5 kcal/g, 단백질 비율 > 30% | 칼로리 밀도 > 2.5 kcal/g |
| BULK_UP       | 칼로리 밀도 높음, 단백질 절대량 > 20g   | 칼로리 밀도 낮음        |
| LEAN_MASS_UP  | 단백질 비율 > 35%, 적정 칼로리          | 지방 비율 > 40%         |

---

### 3-3. Disease Compliance Score (최대 15점)

기본 15점에서 질환 제약 위반 시 감점한다.

| 조건                             | 감점 |
| -------------------------------- | ---- |
| 당뇨 + 당류 비율 > 30%          | -5   |
| 고혈압 + 나트륨 > 600mg/1회     | -5   |
| 고지혈증 + 포화지방 > 5g/1회    | -5   |
| 신장질환 + 단백질 > 15g/1회     | -5   |
| 신장질환 + 칼륨 > 300mg/1회     | -3   |
| 갑상선질환 + 요오드/셀레늄 함유 | +3   |
| 통풍 + 고퓨린                   | 제외 (하드 필터) |

---

### 3-4. Preference Score (최대 10점)

기본 5점에서 시작하여 가감한다.

| 조건                                   | 점수 변동 |
| -------------------------------------- | --------- |
| dislikedFoods에 포함                   | → 0점     |
| preferredFoods에 정확히 일치           | +5        |
| preferredFoods와 카테고리/부분 매칭    | +3        |
| 해당 없음                              | 5점 유지  |

---

### 3-5. Feedback Score (최대 10점)

기본 5점에서 시작하여 과거 피드백 이력에 따라 가감한다.

| feedbackType | 점수 변동 |
| ------------ | --------- |
| like         | +3        |
| saved        | +5        |
| dislike      | -5        |
| ignored      | -1        |

---

## 5단계: 추천량 계산

가장 부족한 영양소를 기준으로 음식의 섭취량을 결정한다.

1. 가장 부족 비율이 높은 영양소를 primary nutrient로 선정
2. 해당 영양소의 부족량을 채우는 데 필요한 음식량 계산
3. 기본 제공량의 0.5배 ~ 2.0배 범위로 클리핑

---

## 6단계: 추천 사유 생성

각 영양소별 기여율과 음식 특성을 기반으로 최대 3개의 사유 태그를 생성한다.

| 조건                                           | 사유 태그         |
| ---------------------------------------------- | ----------------- |
| 단백질 기여율 > 30%                            | "단백질 보충"     |
| 탄수화물 기여율 > 30%                          | "탄수화물 보충"   |
| 지방 기여율 > 30%                              | "지방 보충"       |
| 칼로리 밀도 < 1.2 kcal/g                       | "저칼로리"        |
| 칼로리 밀도 > 2.0 kcal/g (BULK_UP 목표 시)     | "고칼로리"        |
| 식이섬유 > 5g/1회                              | "식이섬유 풍부"   |

## 전체 파이프라인

```
[Spring이 넘긴 데이터]
     │
     ▼
1단계: 끼니별 목표 계산
         mealPattern + mealType → meal_target
     │
     ▼
2단계: 후보 필터링 (foods DB)
         ├── 하드 필터: 알레르기 제외, 고퓨린 제외, 비선호 제외
         └── 후보 리스트 확정
     │
     ▼
3단계: 스코어링 (후보 전체)
         ├── Gap Match Score     (50점)
         ├── Goal Score          (15점)
         ├── Disease Score       (15점)
         ├── Preference Score    (10점)
         └── Feedback Score      (10점)
     │
     ▼
4단계: 정렬 → 상위 N개 → 추천량 계산 → 사유 생성
     │
     ▼
[FastAPI 응답 반환]
```

## 식품 DB 필요 컬럼

추천 알고리즘이 정상 동작하기 위해 foods 테이블에 필요한 컬럼:

| 컬럼            | 타입    | 용도                           | 필수 |
| --------------- | ------- | ------------------------------ | ---- |
| id              | Integer | PK                             | O    |
| name            | String  | 음식명                         | O    |
| category        | String  | 음식 카테고리 (선호도 매칭)    | O    |
| weight          | Float   | 1회 기본 제공량 (g)            | O    |
| calories        | Float   | 칼로리 (kcal)                  | O    |
| protein         | Float   | 단백질 (g)                     | O    |
| carbs           | Float   | 탄수화물 (g)                   | O    |
| fat             | Float   | 지방 (g)                       | O    |
| sodium          | Float   | 나트륨 (mg) — 고혈압 필터      | O    |
| saturated_fat   | Float   | 포화지방 (g) — 고지혈증 필터   | △    |
| sugar           | Float   | 당류 (g) — 당뇨 필터           | △    |
| dietary_fiber   | Float   | 식이섬유 (g) — 당뇨 가점       | △    |
| potassium       | Float   | 칼륨 (mg) — 신장질환 필터      | △    |
| purine_level    | Enum    | 퓨린 수준 (HIGH/MED/LOW) — 통풍 | △    |
| iodine          | Float   | 요오드 (mcg) — 갑상선 가점     | △    |
| selenium        | Float   | 셀레늄 (mcg) — 갑상선 가점     | △    |

## 추가 수집이 필요한 사용자 데이터

현재 온보딩 엔티티에 없지만, 알고리즘 정확도를 높일 수 있는 데이터:

| 데이터                | 용도                        | 비고                                         |
| --------------------- | --------------------------- | -------------------------------------------- |
| 알레르기 상세 식재료  | 특정 식재료 하드 필터       | 현재 diseases에 ALLERGY만 있고 상세 항목 없음 |

> 알레르기 상세는 온보딩에서 수집하거나, 추천 설정(constraints.allergies)을 활용하여 해결 가능
