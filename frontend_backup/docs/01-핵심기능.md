# NutriAgent Frontend 핵심 기능

## 1. 식단 기록 (Meal Recording)

### 음식 검색

- 실시간 검색 (디바운싱 적용)
- 검색 결과 리스트 표시
  - 음식명
  - 1회 제공량
  - 칼로리 및 탄단지(탄수화물, 단백질, 지방)
- 검색 결과 선택 → 식단 저장 화면으로 이동

### 이미지 기반 음식 입력

- 카메라 촬영 / 갤러리 선택
- 이미지 미리보기
- 로딩 상태 표시 (OCR/분류 처리 중)
- 인식 결과 표시
  - OCR 성공: 음식명 + 섭취량 자동 입력
  - 이미지 분류: 신뢰도에 따른 UI
    - ≥80%: "✅ 김치찌개 (신뢰도 85%)" + [맞아요] 버튼
    - 70-79%: "⚠️ 김치찌개 (신뢰도 75%)" + 다른 후보 제시
    - <70%: 상위 3개 후보 선택 + [직접 검색] 버튼
- 인식 실패 시 → 음식 검색으로 이동

### 식단 저장

- 선택된 음식 정보 표시
- 섭취량 입력 (숫자 키패드)
  - 실시간 영양소 계산 및 표시
  - 예상 칼로리, 탄단지 표시
- 식사 시간대 선택 (아침/점심/저녁/간식)
  - 버튼 그룹 또는 세그먼트 컨트롤
- 날짜 선택
  - 기본값: 오늘
  - 좌우 화살표 또는 달력 피커
- [저장] 버튼
- 저장 완료 → 토스트 메시지 + 일간 조회 화면으로 이동

### 식단 조회

- **헤더**
  - 날짜 선택 (좌우 화살표)
  - "오늘" 버튼 (빠른 이동)

- **영양소 요약 카드**
  - 총 칼로리
  - 탄수화물, 단백질, 지방 (그램)
  - 목표 대비 달성률 (프로그레스 바 + 수치(g))
    - 단백질: 초록색
    - 탄수화물: 파란색
    - 지방: 주황색
    - 달성 시: 빨간색

- **시간대별 식단 목록**
  - 아침 섹션
    - 음식 아이템 카드
      - 음식명
      - 섭취량
      - 칼로리 및 탄단지
      - [...] 메뉴 (수정/삭제)
  - 점심 섹션
  - 저녁 섹션
  - 간식 섹션

- **빈 상태**
  - "아직 기록된 식단이 없습니다"
  - [+ 식단 추가하기] 버튼

## 2. 식단 추천 (Meal Recommendation)

### 추천 조회 화면

- **현재 상태 요약**
  - 오늘 섭취한 칼로리/탄단지
  - 목표 대비 부족/초과량
  - 도넛 차트 또는 진행률 바

- **영양소 갭 시각화**
  - 바 차트: 탄수화물, 단백질, 지방 갭
  - 부족한 영양소 강조 표시

- **AI 코칭 메시지**
  - "단백질이 64.5g 부족해요. 닭가슴살 샐러드를 추천드려요!"
  - 말풍선 스타일 또는 카드

- **추천 음식 리스트**
  - 음식 카드 (최대 10개)
    - 음식명
    - 추천 섭취량
    - 예상 칼로리/탄단지
    - 추천 점수 (⭐ 85.5점)
    - 추천 사유 태그 (#단백질보충 #저칼로리)
    - [저장하기] 버튼
  - 슬라이더 또는 그리드 레이아웃

### 추천 설정

- 알레르기 음식 선택
- 선호/비선호 음식 설정
- 저장 버튼

### 피드백

- 추천 음식 카드에 👍/👎 버튼
- 클릭 시 피드백 제출
- "피드백 감사합니다!" 토스트

## 3. 온보딩 (Onboarding)

### Step 1: 환영 화면

- "NutriAgent에 오신 것을 환영합니다!"
- 서비스 소개
- [시작하기] 버튼

### Step 2: TDEE 계산

- 성별 선택 (남/여)
- 나이 입력
- 체중 입력 (kg)
- 신장 입력 (cm)
- 활동량 선택
  - 거의 안 함
  - 가벼운 활동
  - 보통 활동
  - 활발한 활동
  - 매우 활발한 활동
- 자동 TDEE 계산 및 표시

### Step 3: 목표 설정

- 목표 칼로리 (TDEE 기반 추천)
- 목표 탄수화물, 단백질, 지방
  - 슬라이더 또는 숫자 입력
  - 비율 표시 (예: 탄 50%, 단 25%, 지 25%)
- [완료] 버튼 → 메인 화면으로

## 4. 대시보드 / 홈 화면

### 오늘의 요약

- 오늘 날짜
- 총 칼로리 / 목표 칼로리
- 큰 프로그레스 서클

### 빠른 액션

### 최근 식단

- 평균 영양소 섭취량

## 6. UI/UX 원칙

### 색상 팔레트

- **Primary**: 초록색 계열 (건강, 성장)
- **Secondary**: 오렌지 계열 (에너지, 비타민)
- **Accent**: 파란색 (신뢰)
- **Danger**: 빨간색 (초과, 경고)
- **Success**: 녹색 (목표 달성)

### 타이포그래피

- 제목: Bold, 큰 사이즈
- 숫자 (칼로리, 영양소): Monospace, 강조
- 본문: 읽기 편한 Sans-serif

### 인터랙션

- 버튼 클릭: 즉각적인 피드백 (ripple effect)
- 로딩: Skeleton UI 또는 스피너
- 에러: 명확한 에러 메시지 + 재시도 버튼
- 성공: 토스트 메시지 또는 체크 애니메이션

### 접근성

- 충분한 터치 영역 (최소 44x44px)
- 색상에만 의존하지 않는 정보 전달
- 스크린 리더 지원 (aria-label)
